## _기획한 웹 페이지 소개_

- **캘린더처럼 한눈에 상태를 확인할 수 있도록 구성하고, 어렵지 않은 구조와 간단한 수정 방식으로 쉽게 사용할 수 있는 태스크 관리 화면을 기획하였습니다.**

```txt
① 프로젝트 시작 단계에서 기능 구현에 앞서 기본 제공된 디자인을 
   피그마를 활용해 전체 화면 구조와 사용자 흐름을 정리

② 기존 디자인과 다른 구조로 팀"401호"만의 디자인으로 구상

③ 칸반 보드 특성상 작업 상태를 한눈에 파악하는 것이 중요하다고 판단하여,
   Todo / Doing / Done 상태를 좌우로 배치한 3단 보드 구조를 기본 레이아웃으로 구성

④ 페이지 진입 시 전체 진행 상황을 바로 확인할 수 있도록
   통계 영역을 상단에 분리 배치하는 방향으로 시안을 설계

⑤ 수정 시 모달은 아래에 배치하여 한눈에 파악할 수 있으며,
   최소한의 움직임으로도 수정 가능하게 사용자의 편의를 생각하며 구상

⑥ 프로필 변경 시 불편함이 없는 변경 방법과 한눈에 파악할 수 있는,
   클릭 형태로 구상 

⑦ 간결한 카드 구조, 파스텔 톤 컬러, 명확한 영역 구분, 투명도, 
   그리고 반응형 환경에서도 동일한 정보와 흐름 유지를 기준으로 화면을 구성

⑧ 다크모드 / 라이트 모드 구상 시 자연스러운 연결성과,
   사용자의 피로와 전체적인 시각적 요소를 생각하며 구상

    ⇒  위 단계뿐 아니라 모든 팀원들과 함께 각각의 아이디어를 내고 조율함으로써 기획과 구조설계, 기능구현을 순차적으로 개발할 수 있었습니다.
```

## _📝기술적 문제와 해결 과정(Trouble Shooting)_

  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;기존 모달의 전역 CSS와 JS 로직을 상속받는 과정에서 Side Effect 발생,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;애플리케이션 내 타 기능들까지 간섭하여 예기치 못한 런타임 오류 발생
  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;전역 스타일을 삭제하고 각 모달의 리소스를 완전히 격리하는 아키텍처 구상,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;충돌을 일으키는 기존과 다른 클로저나 모듈 형태로 재설계하여 외부 간섭 최소화
 
  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;기간 내에 불완전한 코드를 억지로 끼워 맞춰 기능을 억지로 구현하기보다,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;기존 시스템의 가용성을 유지하고 안정성과 결함문제의 가능성을 차단하는 데 우선순위를 높임

<br>

- 필터 기능 추가 시 기존 구조가 깨지는 문제
  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;필터 버튼을 만들 때 디자인(CSS)과 기능(JS)이 똑같은 이름에 너무 꽉 묶여 있어서,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;검색이나 정렬 기능을 새로 넣으려고 하면 기존에 만들어둔 화면이 계속 틀어지는 문제 발생
 
  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;필터링에 필요한 정보들을 따로 모아 하나의 덩어리(객체)로 관리하고,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;전체 목록에서 조건을 전체 목록에서 조건을 **순차적으로 적용하는 구조로 개선**
 
  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;디자인을 수정해도 기능이 망가지지 않게 되어 코드 관리가 편해졌으며,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;기간, 우선순위, 검색어가 동시에 적용되는 깔끔한 필터링 기능 구현

<br>

### 후에 디자인 및 기획 측면에서의 보완점

- 칸반 보드 상호작용 방식의 한계 및 개선 방향
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;현재 할 일의 상태 변경(Todo → Done 등)을 클릭 기반의 버튼 방식으로 구현함<br>
  
  - 분석&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;칸반 보드의 직관적인 특성을 고려했을 때, 카드를 마우스로 직접 끌어 옮기는<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**드래그 앤 드롭(Drag & Drop)** 방식이 UX 측면에서 더 적합하다고 판단

  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;기획 단계의 시간적 제약으로 반영하지 못한 **드래그 라이브러리를 추후 도입**하여,<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;사용자가 업무를 더 매끄럽게 이동시킬 수 있도록 인터랙션 보완 예정

<br>

- 프로필 이미지 업로드 기능 구현 시  
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;프로필 사진 변경 기능 구현 시 이미지 용량 초과에 대한 기본 제한 로직만 적용함<br>
  
  - 분석&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;고화질 이미지가 일반적인 환경을 충분히 고려하지 못해 사용자 편의성이 떨어짐을 인지<br>
  
  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;사진을 올리기 전 **자동으로 크기를 줄이고 용량을 압축하는 기능을 추가**
